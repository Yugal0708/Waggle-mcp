# Code of Conduct

This project aims to be direct, technical, and respectful.

## Expected behavior

- Be specific about problems, reproductions, and tradeoffs.
- Assume good intent, especially from first-time contributors.
- Critique code, design, and evidence rather than the person.
- Keep review feedback actionable.
- Respect security boundaries and report vulnerabilities privately.

## Unacceptable behavior

- Harassment, insults, or personal attacks.
- Discriminatory language or exclusionary behavior.
- Deliberate spam, derailment, or bad-faith issue filing.
- Publishing private or sensitive information without consent.

## Enforcement

Maintainers may edit, lock, hide, or remove content that makes the project harder to use or maintain. Repeated or severe violations may lead to temporary or permanent bans from project spaces.

## Reporting

- Security issues: follow [`SECURITY.md`](./SECURITY.md)
- General conduct concerns: open a private GitHub maintainer contact if available, or email the project maintainer directly
