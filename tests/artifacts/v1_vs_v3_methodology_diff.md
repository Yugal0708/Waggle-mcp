# v1 vs v3 Methodology Diff

`v1` is the retrieval-only path in [src/waggle/longmemeval_benchmark.py](/Users/abhigyanshekhar/Desktop/MCP/src/waggle/longmemeval_benchmark.py). `v3` is the retrieval-only path built from [src/waggle/memory_benchmark.py](/Users/abhigyanshekhar/Desktop/MCP/src/waggle/memory_benchmark.py) plus `MemoryGraph.query()` in [src/waggle/graph.py](/Users/abhigyanshekhar/Desktop/MCP/src/waggle/graph.py). Both use the same embedding model by default, `all-MiniLM-L6-v2`, and both ultimately score LongMemEval against the same gold support IDs from `longmemeval_s_cleaned.json`. The metric definitions are also aligned at the report level: `R@5` means at least one gold support ID appears in the top 5 retrieved IDs, and `Exact@5` means all gold support IDs are present in the top 5.

The retrieval algorithms are not identical. `v1` prepares one `PreparedLongMemEvalSession` per haystack session, embeds the full session text once, then creates a small synthetic chunk set per session: one full-session chunk plus up to four per-line chunks. In `graph_raw`, it ranks sessions by a fused score combining semantic similarity, lexical overlap, temporal alignment, term coverage, and quoted-phrase overlap, then applies a chunk-derived session boost. In `graph_hybrid`, it reranks the top raw candidates with heuristic reciprocal-rank fusion and chunk-based reranking. So although the reported IDs are session IDs, `v1` is explicitly chunk-aware during ranking even before the final top-5 session list is formed.

`v3` indexes each case into `MemoryGraph` as a graph of session-anchor nodes plus per-turn nodes. It adds `part_of` edges from turns to anchors, `relates_to` edges from anchors back to turns, local bidirectional turn adjacency, and bidirectional adjacency between neighboring session anchors. Retrieval runs through `MemoryGraph.query()` in graph mode with `max_depth=1` and `expand_depth=1`, scoped to the case project. The graph ranker uses embedding similarity, lexical scoring, temporal handling, replay-session signals, graph expansion, node degree, and edge weights over these indexed nodes. The final report then maps the ranked nodes back to unique `session_id` values and scores those session IDs against LongMemEval gold. This means `v3` measures turn/anchor-node retrieval with session-level postprocessing, while `v1` measures direct session ranking with chunk-derived signals mixed into the session score. The comparison is directionally useful, but it is not perfectly apples-to-apples because the internal retrieval unit differs: `v1` ranks sessions with chunk-aware features; `v3` ranks graph nodes and then collapses them to sessions afterward.
