# Support

Use the right channel so issues are actionable:

- Bugs: open a GitHub issue with reproduction steps.
- Feature requests: open a GitHub issue with the use case and proposed behavior.
- Documentation gaps: open a docs issue and point to the exact page or file.
- Security reports: follow [`SECURITY.md`](./SECURITY.md) and do not file public issues for exploitable vulnerabilities.

Before opening an issue:

- Run `waggle-mcp doctor`
- Re-test with `WAGGLE_MODEL=deterministic` if the issue might involve model downloads or local embeddings
- Check [`docs/install/troubleshooting.md`](./docs/install/troubleshooting.md)
- Check [`CONTRIBUTING.md`](./CONTRIBUTING.md) for local dev and test commands
